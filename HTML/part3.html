HTML file Path & Boilerplate code

File path in HTML:

HTML file paths specify the location of files or resources that a webpage needs to access, such as images, videos, scripts, or other HTML documents. These paths tell the web browser where to find the files required to display the content correctly or to execute scripts as intended.
To insert a file in a web page, its source must be known. For example, the syntax ≪img src=" " alt=" "> is used to insert an image file, where the path of the file is mentioned in the source (src).

Types of File Paths:

There are two main types of HTML File Paths:

  1. Absolute Path
  2. Relative Path

1.Absolute Path:

Point directly to a resource's location on the internet and include the full URL, which consists of the protocol (http:// or https://), domain, and path to the resource.
Best for resources that are hosted externally. The browser knows exactly where to find them regardless of the current document’s location.

example


<img src="https://media.geeksforgeeks.org/wp-content/uploads/geek.png" 
     alt="My Image" 
     style="width: 400px" />

output:

2.Relative Path

Specify the path to a resource in relation to the location of the HTML file currently being viewed.
Ideal for resources within the same website. Keeps your HTML portable if the domain changes since the path doesn’t need to be updated.

Example:


<h2>File present in the same folder</h2>
<img src="assets/full.jpeg" 
     alt="My Image" 
     style="width:400px">

output

File present in the same folder

What is and Why we use BIOLERPLATE Code in html

HTML boilerplate code refers to the minimal, essential structure required for any valid HTML document. It is a pre-written template that includes fundamental elements and declarations, providing a starting point for web development projects.

Tag Purpose Description
<!DOCTYPE html> Document declaration Declares the document type and version of HTML (HTML5).
<html lang="en"> Root element Wraps all content and specifies the document language.
<head> Metadata container Holds non-visible info like encoding, title, and links.
<meta charset="UTF-8"> Character encoding Ensures support for a wide range of characters.
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Responsive design Configures the viewport for mobile-friendly layout.
<title>Document Title</title> Page title Sets the title shown in the browser tab.
<link rel="stylesheet" href="style.css"> External CSS Links to a stylesheet for styling the page.
<body> Visible content Contains everything the user sees on the page.
<script src="script.js"></script> External JavaScript Links to a JS file for interactivity and dynamic behavior.

Div Tag

The <div> tag, used in over 90% of web pages, divides content into sections and serves as a flexible container for text, images, and links. Easily styled with CSS and customized using classes or IDs, it often uses percentage-based widths like 50% or 33.33% for responsive layouts, making it key to web page structure.

  • Block-Level Element:The <div tag is a block-level element, meaning it takes up the full width and starts on a new line.
  • Styling with CSS: It can be styled using CSS, allowing customization like width, background color, padding, and margins.
  • Inline-Block Option:By using display: inline-block, a
    can behave like an inline element while retaining block-level features.
  • Used for Layouts:<div> is often used in layouts with Flexbox or Grid to organize content and arrange sections of a webpage.

Class & ID in HTML:

In HTML, id and class are attributes used to identify and group elements, primarily for styling with CSS and manipulating with JavaScript

  • ID Attribute:

    Uniqueness: An id attribute is designed to uniquely identify a single element within an entire HTML document. No two elements on the same page should have the same id.

    Purpose: It is used when you need to target a very specific element for styling or scripting, such as a unique header, a navigation menu, or a specific form input.

    CSS Selector: In CSS, an id is selected using the hash symbol (#) followed by the id value, for example:

    JavaScript: In JavaScript, you can access an element by its id using document.getElementById()
  • Class Attribute:

    Non-uniqueness: A class attribute can be applied to multiple HTML elements across a document.

    Purpose: It is used to group elements that share common characteristics or require similar styling or behavior.For instance, you might have a class="city" to apply the same styling to multiple city names.

    Multiple Classes: An element can have multiple classes assigned to it, separated by spaces, for example: <p class="intro highlight">. CSS Selector: In CSS, a class is selected using a period (.) followed by the class name,

    JavaScript: In JavaScript, you can access elements by their class name using document.getElementsByClassName().

Example of DIV TAG , CLASS & ID


<!DOCTYPE html>
<html>
<head>
    <title>Div Class and ID Example</title>
</head>
<body>
    <div id="header">
        <h1>Welcome to My Website</h1>
    </div>

    <div class="content">
        <p>This is the main content area.</p>
    </div>

    <div class="content">
        <p>Here is another section with the same class.</p>
    </div>

    <div id="footer">
        <p>Footer information goes here.</p>
    </div>
</body>
</html>
OUTPUT:

Welcome to My Website

This is the main content area.

Here is another section with the same class.

Footer information goes here.
